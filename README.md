# LEC — leclaser.com rebuild

A page-for-page **rebuild and major upgrade** of [leclaser.com](https://leclaser.com)
— **LEC**, a precision **laser-cutting / CNC / acrylic-fabrication & signage** company
in Laktaši, Bosnia & Herzegovina (founded 2003). Next.js 14 + Tailwind + framer-motion,
deployed on Vercel. **Language: Bosnian/Serbian/Croatian (Latin, ijekavica).**

> Deploy: import this repository into **Vercel** (auto-detected as Next.js). See **[HANDOFF.md](./HANDOFF.md)**
> for the full owner setup — environment variables, custom domain (`leclaser.com`), and content editing.

## What it is

A high-conversion marketing site with an "engineered / laser" design system
(graphite canvas, frosted-acrylic panels, registration-crosshair + kerf-cut motifs,
single hot laser-orange accent, technical mono labels, cut/reveal motion).

### Pages (78 prerendered)
- **Home** `/` — animated blueprint hero, services, process, portfolio, why-us, locations teaser, FAQ.
- **Usluge** `/usluge` + 6 service detail pages (`/usluge/[slug]`).
- **Portfolio** `/portfolio` + 6 category pages (`/portfolio/[slug]`).
- **Lokacije** `/lokacije` (filterable browser) + **52 city pages** (`/lokacije/[grad]`)
  across BiH, Srbija, Hrvatska, Crna Gora — each with unique, locally-grounded copy.
- **O nama** `/o-nama` — interactive 2003→2023 timeline.
- **Kontakt** `/kontakt` — quote form + map + phone/WhatsApp/Viber.
- `politika-privatnosti`, `404`, `sitemap.xml`, `robots.txt`, `opengraph-image`, `manifest`.

### Conversion (CRO)
- Sticky header with phone + "Zatraži ponudu" always visible.
- Floating mobile dock: Call + WhatsApp + Viber.
- Quote form on every key page (prefilled with service/city), honeypot-protected.
- Localized CTAs + FAQ on every city page; trust signals (od 2003, prvi u BiH, 4 zemlje).
- Route progress bar + `loading.tsx` skeletons so navigation never feels dead.

### SEO
- Per-page canonical, OG/Twitter, BCS metadata.
- JSON-LD: Organization, LocalBusiness, Service, BreadcrumbList, FAQPage (+ per-city LocalBusiness with `areaServed`).
- `sitemap.xml` (all 78 routes), `robots.txt`, dynamic OG image.

## Content pipeline
City / service / portfolio copy was generated by a multi-agent workflow and lives in
`src/data/*.ts` (typed). To regenerate from a workflow output JSON:

```bash
node scripts/build-data.mjs <path-to-workflow-output.json>
```

## Lead delivery (action needed for live forms)
The `/api/lead` route always logs leads to Vercel function logs and works out of the box.
To deliver leads by **email** or to a **CRM/webhook**, set env vars in Vercel:

| Env | Purpose |
|-----|---------|
| `RESEND_API_KEY` | Send lead emails via Resend |
| `LEAD_TO` | Destination inbox (defaults to `info@leclaser.com`) |
| `LEAD_FROM` | Verified sender (e.g. `LEC <upiti@leclaser.com>`) |
| `LEAD_WEBHOOK_URL` | POST each lead to Zapier/Make/n8n/GHL |

> Phone / WhatsApp / Viber CTAs are fully live now — the form is the only channel that needs an env to deliver.

## Deploy
This project deploys on **Vercel**. Recommended: Git-connected auto-deploy.

1. In Vercel: **Add New → Project → Import** this Git repository.
2. Framework preset is auto-detected (**Next.js**) — no build settings to change.
3. Add the environment variables (see **[HANDOFF.md](./HANDOFF.md)**) and click **Deploy**.

Every push to the default branch then deploys automatically. Full owner guide
(env vars, `leclaser.com` domain, editing content) is in **[HANDOFF.md](./HANDOFF.md)**.
